# 8. GitHub Actions Release Pipeline

Date: 2026-07-14

## Status

Accepted

## Context

As the monorepo grows, we need a reliable, automated way to build and distribute compiled release artifacts (installers and binaries) for the `mkv-subtitle-converter` Tauri application. The official `actions/create-release` action was deprecated and archived by GitHub, requiring us to choose a community standard alternative. Furthermore, the monorepo structure introduces complexities around triggering releases without kicking off builds for unrelated sibling projects.

## Decision

We will implement a GitHub Actions workflow to automate the release process with the following architectural choices:

1.  **Community Action**: We will use `softprops/action-gh-release` to handle the creation of GitHub Releases and the uploading of assets, as it is the most widely adopted and stable community replacement.
2.  **Monorepo Tagging Convention**: To isolate the `mkv-subtitle-converter` release lifecycle from other projects in this monorepo, the workflow will only trigger on tags matching the `mkv-subtitle-converter-v*` pattern (e.g., `mkv-subtitle-converter-v1.1.0`).
3.  **Target Architectures**: The build matrix will explicitly target the default architectures for Linux (`ubuntu-latest`), macOS (`macos-latest` for Apple Silicon/Intel defaults), and Windows (`windows-latest`).
4.  **Installer Artifacts**: Instead of compressing the raw executables with READMEs, the workflow will directly extract and upload the polished installer bundles (e.g., `.msi`, `.exe`, `.dmg`, `.AppImage`, `.deb`) natively generated by Tauri across the respective operating systems.

## Consequences

*   **Positive**: Releases are fully automated, consistent, and strictly isolated per project within the monorepo. End users receive standard platform-specific installers directly attached to the GitHub Release.
*   **Negative/Constraint**: Maintaining the release pipeline adds slightly more complexity, and builds can take time across the GitHub Actions runners due to full-scale compilation and sidecar bundling for each target operating system.
